GraphQL в данном решении значительно улучшает взаимодействие между потребителями (web-приложением и core-app) и сервисом client-info. Рассмотрим основные преимущества:

1. Запрос только необходимых данных

В REST API потребителю приходилось делать несколько запросов для получения информации о клиенте, документах и родственниках.
В GraphQL в одном запросе можно получить всю необходимую информацию, избегая дублирования запросов.
Пример запроса в GraphQL:

query {
  client(id: "123") {
    name
    age
    documents {
      type
      number
    }
    relatives {
      name
      relationType
    }
  }
}

Это значительно снижает нагрузку на сервер и ускоряет работу клиентских приложений.

2. Гибкость запросов без изменения API

В REST API при изменении структуры данных необходимо добавлять новые эндпоинты.
В GraphQL структуру можно расширять без изменения существующего API, просто добавляя новые поля.
Если позже нужно добавить адреса клиента, в REST пришлось бы делать новый эндпоинт /clients/{id}/addresses, а в GraphQL — просто добавить поле addresses в Client.

3. Уменьшение RPS (Requests per Second)

В REST API сценарий требует 3 запроса:
/clients/{id}
/clients/{id}/documents
/clients/{id}/relatives
В GraphQL достаточно 1 запроса, что уменьшает нагрузку на сервер и задержки сети.
4. Улучшенная производительность фронтенда

Web-приложение больше не ждет выполнения нескольких REST-запросов.
Все данные загружаются одним запросом, что ускоряет рендеринг страниц.
